---
title: "Audit API"
description: "Browse, filter, export, and clear the plugin activity log that records user actions and WordPress events."
---

<Note>All audit endpoints are available since v1.4.0.</Note>

## GET /audit

Returns paginated audit log entries in reverse chronological order. Supports filtering by action type, user, and date range.

```bash
curl -X GET "https://example.com/wp-json/wp-manager-pro/v1/audit?page=1&per_page=50" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Parameters

<ParamField path="page" type="integer">
  Page number. Defaults to `1`.
</ParamField>

<ParamField path="per_page" type="integer">
  Results per page. Defaults to `50`, maximum `200`.
</ParamField>

<ParamField path="action_type" type="string">
  Filter by a specific action slug (e.g. `plugin.activated`, `user.login`). Use `GET /audit/action-types` for a list of available values.
</ParamField>

<ParamField path="user_id" type="integer">
  Filter by WordPress user ID.
</ParamField>

<ParamField path="from" type="string">
  Start of the date range in `Y-m-d` format. Entries on or after this date are included.
</ParamField>

<ParamField path="to" type="string">
  End of the date range in `Y-m-d` format. Entries on or before this date are included.
</ParamField>

### Response

<ResponseField name="logs" type="array">
  <Expandable title="log entry properties">
    <ResponseField name="id" type="integer">Log entry ID.</ResponseField>
    <ResponseField name="user_id" type="integer">ID of the user who performed the action. `0` for system events.</ResponseField>
    <ResponseField name="user_name" type="string">Login name of the user, or `system` for automated events.</ResponseField>
    <ResponseField name="action" type="string">Action slug (e.g. `snippet.created`, `backup.created`).</ResponseField>
    <ResponseField name="object_type" type="string">The type of object affected (e.g. `plugin`, `snippet`, `backup`).</ResponseField>
    <ResponseField name="object_name" type="string">Name or identifier of the affected object.</ResponseField>
    <ResponseField name="extra" type="string | null">JSON-encoded extra context, or `null`.</ResponseField>
    <ResponseField name="ip_address" type="string">IP address of the request.</ResponseField>
    <ResponseField name="created_at" type="string">Timestamp of the event.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="total" type="integer">Total number of matching log entries.</ResponseField>
<ResponseField name="page" type="integer">Current page number.</ResponseField>
<ResponseField name="per_page" type="integer">Results per page.</ResponseField>
<ResponseField name="total_pages" type="integer">Total number of pages.</ResponseField>

## DELETE /audit/clear

Permanently deletes all audit log entries.

<Warning>
  This truncates the entire audit log table. All activity history is permanently removed.
</Warning>

```bash
curl -X DELETE https://example.com/wp-json/wp-manager-pro/v1/audit/clear \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">Always `true` on success.</ResponseField>

## POST /audit/export

Exports all audit log entries as a CSV file and stores the result in a short-lived transient (120 seconds). Returns a signed download URL.

```bash
curl -X POST https://example.com/wp-json/wp-manager-pro/v1/audit/export \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">Always `true`.</ResponseField>
<ResponseField name="download_url" type="string">A time-limited URL to download the CSV export. Expires after 120 seconds.</ResponseField>

---

## GET /audit/download

Streams the exported CSV generated by `POST /audit/export`. The response is a CSV file download, not JSON.

<Note>
  Use the `download_url` returned by `POST /audit/export` directly — it includes the required `key` and `_wpnonce` query parameters.
</Note>

```bash
curl -o audit-log.csv "https://example.com/wp-json/wp-manager-pro/v1/audit/download?key=KEY&_wpnonce=NONCE"
```

### Parameters

<ParamField path="key" type="string" required>
  The one-time download key returned by `POST /audit/export`.
</ParamField>

## GET /audit/action-types

Returns a list of all distinct action type slugs present in the current audit log. Use these values as the `action_type` filter parameter for `GET /audit`.

```bash
curl -X GET https://example.com/wp-json/wp-manager-pro/v1/audit/action-types \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="types" type="array">Sorted array of distinct action slug strings present in the log (e.g. `["backup.created", "plugin.activated", "user.login"]`).</ResponseField>
