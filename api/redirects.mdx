title: "Redirects API"
description: "Create and manage 301, 302, and 307 URL redirects, and import or export redirect rules as CSV files."
---

<Note>All redirect endpoints are available since v1.4.0.</Note>

## GET /redirects

Returns a paginated list of redirects. Supports full-text search on source and target URLs.

```bash
curl -X GET "https://example.com/wp-json/wp-manager-pro/v1/redirects?page=1&per_page=50" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Parameters

<ParamField path="page" type="integer">
  Page number. Defaults to `1`.
</ParamField>

<ParamField path="per_page" type="integer">
  Results per page. Defaults to `50`, maximum `200`.
</ParamField>

<ParamField path="search" type="string">
  Filter results to redirects whose source or target contains this string.
</ParamField>

### Response

<ResponseField name="redirects" type="array">
  <Expandable title="redirect properties">
    <ResponseField name="id" type="integer">Redirect ID.</ResponseField>
    <ResponseField name="source" type="string">Source URL path (always starts with `/`).</ResponseField>
    <ResponseField name="target" type="string">Destination URL or path.</ResponseField>
    <ResponseField name="type" type="integer">HTTP redirect code: `301`, `302`, or `307`.</ResponseField>
    <ResponseField name="hits" type="integer">Number of times this redirect has fired.</ResponseField>
    <ResponseField name="created_at" type="string">Creation timestamp.</ResponseField>
    <ResponseField name="updated_at" type="string">Last-updated timestamp.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="total" type="integer">Total number of redirects.</ResponseField>
<ResponseField name="page" type="integer">Current page number.</ResponseField>
<ResponseField name="per_page" type="integer">Results per page.</ResponseField>
<ResponseField name="total_pages" type="integer">Total number of pages.</ResponseField>

## POST /redirects

Creates a new redirect rule. The source path is automatically normalised to start with `/`.

```bash
curl -X POST https://example.com/wp-json/wp-manager-pro/v1/redirects \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"source": "/old-page", "target": "/new-page", "type": 301}'
```

### Parameters

<ParamField path="source" type="string" required>
  The URL path to redirect from (e.g. `/old-page`). Must be unique — duplicate sources return `409`.
</ParamField>

<ParamField path="target" type="string" required>
  The destination URL or path. Relative paths are converted to absolute using the site's home URL at runtime.
</ParamField>

<ParamField path="type" type="integer">
  HTTP redirect status code. Must be `301`, `302`, or `307`. Defaults to `301`.
</ParamField>

### Response

Returns HTTP `201` on success.

<ResponseField name="success" type="boolean">Always `true`.</ResponseField>
<ResponseField name="redirect" type="object">The newly created redirect object with all fields.</ResponseField>

## PUT /redirects/{id}

Updates an existing redirect.

```bash
curl -X PUT https://example.com/wp-json/wp-manager-pro/v1/redirects/12 \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"target": "/updated-destination", "type": 302}'
```

### Parameters

<ParamField path="source" type="string">
  Updated source path.
</ParamField>

<ParamField path="target" type="string">
  Updated destination URL or path.
</ParamField>

<ParamField path="type" type="integer">
  Updated HTTP code: `301`, `302`, or `307`.
</ParamField>

### Response

<ResponseField name="success" type="boolean">Always `true`.</ResponseField>
<ResponseField name="redirect" type="object">The updated redirect object with all fields.</ResponseField>

## DELETE /redirects/{id}

Permanently deletes a redirect rule.

```bash
curl -X DELETE https://example.com/wp-json/wp-manager-pro/v1/redirects/12 \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">Always `true` on success.</ResponseField>

## POST /redirects/export

Exports all redirects as a CSV file and stores the result in a short-lived transient (120 seconds). Returns a signed download URL.

```bash
curl -X POST https://example.com/wp-json/wp-manager-pro/v1/redirects/export \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">Always `true`.</ResponseField>
<ResponseField name="download_url" type="string">A time-limited URL to download the CSV file. Expires after 120 seconds.</ResponseField>

---

## GET /redirects/download

Streams the exported CSV file generated by `POST /redirects/export`. The response is a CSV file download, not JSON.

<Note>
  Use the `download_url` from `POST /redirects/export` directly — it includes the required `key` and `_wpnonce` query parameters.
</Note>

```bash
curl -o redirects.csv "https://example.com/wp-json/wp-manager-pro/v1/redirects/download?key=KEY&_wpnonce=NONCE"
```

### Parameters

<ParamField path="key" type="string" required>
  The one-time download key returned by `POST /redirects/export`.
</ParamField>

---

## POST /redirects/import

Imports redirects from an uploaded CSV file. The first row is treated as a header and skipped automatically. Duplicate source paths are silently ignored.

```bash
curl -X POST https://example.com/wp-json/wp-manager-pro/v1/redirects/import \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -F "file=@redirects.csv"
```

### Parameters

<ParamField path="file" type="file" required>
  A CSV file with columns: `Source`, `Target`, `Type` (HTTP code). The `Type` column defaults to `301` if omitted or invalid.
</ParamField>

### Response

<ResponseField name="success" type="boolean">Always `true`.</ResponseField>
<ResponseField name="imported" type="integer">Number of redirects successfully imported.</ResponseField>
